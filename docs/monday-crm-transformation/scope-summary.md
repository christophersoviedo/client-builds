## Project Objective

Design and implement a Monday.com CRM operating system that supports the client's sales workflow from lead intake through closed revenue and post-sale handoff.

The CRM should give the team:

- A clear source of truth for contacts, accounts, deals, activities, and pipeline status.
- Standardized sales stages and ownership.
- Practical automations that reduce manual follow-up work.
- Dashboards that show pipeline health, sales activity, conversion, and revenue movement.
- Integrations that reduce duplicate data entry across forms, email, calendar, marketing, and operational tools.

## Working Assumptions

- The client wants Monday.com to function as the central CRM, not only as a task tracker.
- The sales workflow includes some version of lead intake, qualification, active opportunity management, proposal or quote follow-up, closed-won handoff, and closed-lost tracking.
- The CRM will need multiple connected boards rather than a single overloaded board.
- The client likely needs both pipeline visibility and activity accountability.
- Existing client data, tools, roles, and workflow rules have not yet been provided.

## Key Risks

- Building around unconfirmed sales stages can create rework.
- Automating before data standards are defined can multiply inconsistent records.
- Overloading one board with every CRM object can reduce reporting accuracy.
- Integrations may be constrained by plan level, available APIs, authentication access, and third-party tool permissions.
- Dashboard KPIs may be misleading if source columns are not consistently required and maintained.

## Recommended Next Step

Run a CRM discovery session before implementation. The session should confirm sales process, board ownership, required fields, reporting requirements, existing tools, data migration needs, and integration priorities.
